R Naming Conventions

R programming language follows its own set of naming conventions. Here are the key points:

Variables and Functions

  • Use lowercase letters for variable names.
  • Separate words with underscores or use camelCase.
  • Variable names should be descriptive and meaningful.
  • Functions should have descriptive names that indicate their purpose.
  • Avoid using reserved words as variable names.

Constants

  • Constants are not explicitly defined in R, but conventionally use uppercase letters for variables that are not intended to be modified.

Classes

  • R uses S3 and S4 class systems for object-oriented programming.
  • Class names should be meaningful and follow the same conventions as variable names.

Packages

  • Package names should be descriptive and lowercase.
  • Avoid using names that conflict with existing R packages.
